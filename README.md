# Candidate Viewer

HR-инструмент для просмотра списка кандидатов и детальной информации по каждому.

## Запуск

```bash
npm install
npm run dev        # локальный сервер разработки → http://localhost:5173
npm run build      # продакшен сборка
npm run preview    # предварительный просмотр сборки
```

## Стек

- **React 18** + **TypeScript** (строгий режим, без `any`)
- **Vite** — сборка
- **React Router v6** — маршрутизация
- **Tailwind CSS v3** — стилизация

## Реализованные задания

### Задание 1 — Список кандидатов
- Загрузка через `services/api.ts` с имитацией задержки 500 мс (`setTimeout`)
- Состояния loading / error / empty state
- Карточки: ФИО, город, опыт, стек, вердикт с цветовой индикацией

### Задание 2 — Поиск и фильтрация
- Поиск по ФИО (регистронезависимый)
- Фильтр по вердикту (ПОДХОДИТ / ЧАСТИЧНО / НЕ ПОДХОДИТ / Все)
- Счётчик найденных кандидатов
- Поиск и фильтр работают совместно
- **Бонус**: debounce 300 мс через кастомный хук `useDebounce`
- **Бонус**: параметры поиска/фильтра/сортировки сохраняются в URL (query params)

### Задание 3 — Детальная страница `/candidate/:id`
- Полная информация: контакты, стек, образование
- Таблица опыта работы
- Критерии оценки с иконками (✓ / ⚠ / ✗) и цветами
- Вопросы для собеседования + резюме HR
- Кнопка «Назад к списку»
- 404-обработка несуществующего id

### Задание 4 — Вёрстка
- Семантический HTML (`<header>`, `<main>`, `<section>`, `<nav>`, `<table>`)
- Адаптивная сетка: 1 колонка (мобильный) → 2–3 колонки (десктоп)
- Без горизонтального скролла на 375px
- aria-label и aria-hidden для доступности

### Задание 5 — Дополнительно (всё реализовано)
- Сортировка по дате / имени / опыту
- Бейдж статуса кандидата (new / reviewing / interviewed)
- Кастомные хуки: `useDebounce`, `useCandidates`
- URL persistence (query params) для поиска, фильтра и сортировки

## Структура проекта

```
src/
├── components/
│   ├── CandidateCard/      — карточка в списке
│   ├── CriteriaList/       — критерии оценки с иконками
│   ├── SearchBar/          — поиск + фильтр + сортировка
│   ├── StatusBadge/        — бейдж статуса кандидата
│   ├── VerdictBadge/       — бейдж вердикта с цветом
│   └── UI/                 — Spinner
├── hooks/
│   ├── useDebounce.ts      — debounce 300 мс
│   └── useCandidates.ts    — загрузка данных (loading/error)
├── pages/
│   ├── CandidatesPage.tsx  — главная страница ("/")
│   └── CandidateDetailPage.tsx — детальная страница ("/candidate/:id")
├── services/
│   └── api.ts              — fetchCandidates() с кэшем и задержкой
├── types/
│   └── candidate.ts        — TypeScript-типы данных
├── App.tsx
└── main.tsx
```
